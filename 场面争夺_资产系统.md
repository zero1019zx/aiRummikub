# 场面争夺 — 资产系统说明(Boss / 遗物 / 技能)

> 目标:把 Boss、遗物、技能三类内容做成**数据驱动、可维护、可平衡优化的组件**。
> 内容(数值/机制)放在 `game/data/*.json` = "资产";代码只负责加载与执行 = "组件"。**改内容改 JSON,不碰逻辑。**

## 0. 分层与文件

| 层 | 文件 | 职责 |
|---|---|---|
| 资产(数据) | `game/data/bosses.json` | Boss/精英:血量、阶段、机制、美术、文案 |
| 资产(数据) | `game/data/relics.json` | 遗物(被动):效果参数、挂载点、模式、分类 |
| 资产(数据) | `game/data/skills.json` | 技能(主动):效果、次数、模式 |
| 资产(数据) | `game/data/arena_balance.json` | 全局常量:血量/抽牌/回复/牌型分 |
| 组件(加载) | `game/scripts/arena_db.gd` (`ArenaDB`) | 读 JSON、校验、查询接口 |
| 组件(效果) | `game/scripts/relics.gd` (`RelicManager`) | 遗物挂载点(建议改为读 relics.json) |
| 组件(效果) | 建议新增 `skill_manager.gd` | 技能次数与触发 |

**被动 vs 主动**:遗物=拿到就常驻、自动生效;技能=玩家主动点、每场战斗有次数。UI 上建议拆成两条(现有"遗物条"保留给遗物,血条下再加一条技能条)。

## 1. Boss 资产 (`bosses.json`)

### 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一键 |
| `name` / `intent` / `teaches` | string | 名称 / 出场预告文案 / 它要教玩家什么 |
| `role` | string | tutorial_elite / elite / boss / final_boss(商店与投放分类) |
| `layers` | int[] | 可投放的爬塔层 |
| `art` | string | `assets_v2/avatars/` 下的圆形头像文件名 |
| `hp` | int | 总血量(与 `hp_segments` 二选一) |
| `hp_segments` | int[] | 群战多目标各自血量(如鬣狗 6×40) |
| `gating` | bool | 阶段门:单击不跨阶段血量阈值溢出 |
| `phases` | obj[] | 阶段列表,见下 |

每个 phase:`{"hp_share":0..1, "threat":int, "mechanics":[...], "label":"可选"}`
- `hp_share`:该段占总血比例,**一个 boss 的所有 share 必须合计=1**(ArenaDB 会校验)。
- `threat`:该段每回合对玩家的固定伤害(**单怪难度主旋钮**)。
- `mechanics`:该段生效的机制(见机制目录)。

### 机制目录(authoritative · 含公式)

| type | 参数 | 公式 / 行为 |
|---|---|---|
| `resist` | soft, k | `eff = min(d,soft) + max(0,d-soft)*k`(韧性,局部打折) |
| `armor` | value, pierce_at? | `eff = max(0, d-value)`;若 `d≥pierce_at` 则无视护甲 |
| `cap_hit_pct` | pct | `eff = min(d, 最大血*pct)`(霸体,终Boss身份) |
| `ban_pair` | — | 对子链 → mult 0、base 0 |
| `nullify_shape` | lens[] | 长度∈lens 的链 → 0 |
| `shape_mod` | target, op | target∈{run,group,pair,all};op∈{halve(向下取整),zero,minus1(下限1)} 作用于该类链的牌型分 |
| `dirty_color` | rotate[], telegraph, floor | 当前脏色=rotate按回合轮换;含脏色的链 mult=0;总 mult 保底=floor;telegraph=预告下回合 |
| `freeze_chain` | count, telegraph | 锁定 count 条链:**不可重组、不计新鲜倍率**(锁链本身,勿移牌——见 Boss 文档的实现坑) |
| `charge` | countdown, damage, interrupt, telegraph | 每玩家回合-1;归零释放 damage;当回合伤害≥interrupt 则打断重置 |
| `overflow_reflect` | pct | 若 d>当前血:`玩家受伤 = (d-当前血)*pct` |
| `pollute_deck` | per_turn | 每敌方回合往牌库塞 per_turn 张污泥牌(面值0、不成链) |
| `single_target_no_splash` | — | 伤害只结算一个目标,溢出蒸发(配 hp_segments) |
| `summon` | every, count, cap | 每 every 回合补 count 个目标,上限 cap |

> 新增机制 = 在此表加一行 + 在 `arena_db.gd` 的 `known_mech` 加名字 + 在扣血/算分处实现。

## 2. 遗物资产 (`relics.json`)

### 字段

`id, name, short(图标占位字), icon(assets_v2/relics/下文件名), desc, price, rarity, modes[], category, hook, params{}`

- `modes`:`battle` / `arena`,决定在哪种模式的商店出现与生效。
- `category`:`survival / damage / tempo / economy`(配商店池、平衡时按类看)。
- `hook`:**挂载点**——代码在该时机查询此遗物并套用 `params`。
- `params`:效果数值(**全部可调**,这是把原本硬编码的 -2/0.2/0.5 抽出来的关键)。

### 挂载点目录(hook)

| hook | 现有方法(relics.gd) | params 例 | 模式 |
|---|---|---|---|
| `chips_for_set` | chips_for_set() | color/mult, kind/bonus, ratio | battle |
| `mult_step` | mult_step() | step | battle |
| `enemy_reorg_bonus` | enemy_reorg_bonus() | per_tile | battle |
| `modify_enemy_damage` | modify_enemy_damage() | flat / pct | both |
| `lifesteal` | lifesteal() | ratio, cap_per_turn | both |
| `draw_count` | draw_count() | extra | both |
| `exchange_refund` | exchange_refund() | ratio | both |
| `enemy_draw_blocked` | enemy_draw_blocked() | chance | battle |
| `pair_limit` | pair_limit() | limit | both |
| `interest` | interest() | per, cap | both |
| `regen` | 新增 | flat(加到每回合定额回复) | arena |
| `lethal_save` | 新增 | per_fight, leave_hp(免死) | both |
| `on_phase_change` | 新增 | heal(Boss段切换回血) | arena |
| `shape_score_mod` | 新增 | min_len, bonus(长链牌型分+) | arena |
| `arena_settle` | 新增 | first_chain_mult(首条新鲜链倍率+) | arena |
| `arena_base` | 新增 | per_card(每张出牌base+) | arena |
| `arena_recompose` | 新增 | bonus(重排旧链成新链base+) | arena |
| `auto_cleanse` | 新增 | every, count(自动解debuff) | arena |

**当前 21 个遗物**:12 个沿用(battle)+ 9 个 arena 新增(石肤/再起之风/庇护符/阶段锚/长链宗师/专注新章/重手/重组引擎/净化图腾)。

### 让效果也数据驱动(建议改造)

现在 `relics.gd` 的数值硬编码在方法里(如 `wood_shield` 写死 -2)。建议:`RelicManager` 启动时 `DEFS = ArenaDB.relics_by_id`,方法改为读 params,例:

```gdscript
func modify_enemy_damage(dmg: int) -> int:
    for id in owned:
        var r = DB.relic(id)
        if r.get("hook") == "modify_enemy_damage":
            var p = r.get("params", {})
            if p.has("flat"): dmg -= int(p["flat"])
            if p.has("pct"):  dmg = int(dmg * (1.0 - float(p["pct"])))
    return max(0, dmg)
```

这样调"木盾挡几点"只改 JSON,不动代码。

## 3. 技能资产 (`skills.json`)

### 字段

`id, name, icon, desc, charges(每场次数), modes[], category, effect{type, ...}`

### 效果目录(effect.type)

| type | 参数 | 行为 |
|---|---|---|
| `redraw_hand` | — | 弃光手牌重抽等量(救卡手/排污泥) |
| `extra_draw` | count | 立即多摸 count 张 |
| `cleanse` | targets[] | 解除本回合指定 debuff(freeze_chain/dirty_color) |
| `shield` | value | 本回合获得护盾,先抵敌方伤害 |
| `base_mult` | factor | 本回合 base ×factor(**仍过阶段门,不会一招制敌**) |
| `delay_charge` | turns | Boss 蓄力读条 +turns |

**当前 6 个技能**:重整旗鼓 / 探牌 / 破冰 / 壁垒 / 蓄力一击 / 拖延。
建议:`SkillManager` 持 `{id: 剩余次数}`,每场战斗开始重置为 `charges`;技能条 UI 复用 `relic_bar` 的按钮样式(参 main.gd `_refresh_relic_bar`)。

## 5. 加载组件 `ArenaDB` 接口

```gdscript
var DB := ArenaDB.new()
DB.load_all()                       # 读全部JSON+校验; 返回是否无错
DB.bosses_for_layer(3)              # 第3层可投放的boss
DB.boss_total_hp(b)                 # 总血(兼容hp_segments)
DB.boss_phase_floors(b)             # 阶段门血量边界, 如 cerberus -> [178,89,0]
DB.relics_for_mode("arena","survival")
DB.skills_for_mode("arena")
DB.player_cfg(); DB.shape_score_table()
DB.errors                          # 校验问题列表(开发期)
```

## 6. 接入 main.gd(改动指引)

1. **Boss 投放(仅 arena)**:`_begin_layer` 里若 `arena_mode`,用 `DB.bosses_for_layer(floor_num)` 选 boss,`enemy_hp = DB.boss_total_hp(b)`,缓存 `phases / floors / 当前段索引`。**原 `BATTLE_HP` 与 battle/score 模式不动**(只为 arena 接 boss 表)。
2. **算分处 `_arena_settle`**:计算 mult 时套用当前段的 `shape_mod / dirty_color / ban_pair / nullify_shape / freeze_chain`(包一层在 `_shape_score` 外);算完 `gained` 后、扣血前依次过 `resist/armor/cap_hit_pct`,再做阶段门 `min(d, enemy_hp - floor)`,再 `overflow_reflect`。
3. **回合结束**:玩家 `player_hp = min(max, player_hp + regen + Σ regen遗物)`;不再回满。
4. **遗物**:`RelicManager` 读 `relics.json`(见 §2 改造)。
5. **技能**:加 `SkillManager` + 技能条,触发对应 effect。

## 7. 调参速查(哪个旋钮干什么)

| 想调的东西 | 改哪里 |
|---|---|
| 全局难度(都太难/太易) | `arena_balance.json → regen_per_turn` |
| 单个 Boss 难度 | 该 boss 各段 `threat` |
| 对局长度/节奏 | `hp` 或 `hp_share`、阶段数 |
| 防秒地板 | 阶段数(=最少回合);终Boss `cap_hit_pct` |
| 卡手/爽感 | `draw_per_turn`(全局)、greedy_hand / foresight |
| Boss 花样 | 各段 `mechanics` 参数 |
| 商店强度 | 遗物 `price` / `rarity` / `category` 配池 |
