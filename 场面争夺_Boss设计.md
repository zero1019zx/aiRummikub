# 场面争夺 — Boss / 精英设计文档(v2 · 数据驱动)

> 本版根据补2经济下的真实模拟重写。**去掉了上一版被否决的"全局软上限"**,改用 **阶段门 + 定额回复** 做系统级防秒。
> 所有数值都落在 `game/data/` 的 JSON 资产里(`bosses.json` / `relics.json` / `skills.json` / `arena_balance.json`),本文是设计依据与说明;改平衡改 JSON,不改代码。资产字段规范见《场面争夺_资产系统.md》。

## 0. 公式与经济回顾

- **回合伤害 = base × mult**
  - `base` = 本回合**从手里打出的牌**的面值之和(只算新牌)。
  - `mult` = 本回合**"新鲜链"**的牌型分之和。牌型分:对子1 / 刻子2 / 三连2 / 四连4 / 五连5 / 六连6 / 七连7。
  - "新鲜链" = 这回合改变了组成结构的链(签名变化);已结算且原样没动的链是旧链,倍率算 0,**重排成不同组成才再生效**。牌永不消失,消耗的是倍率。
- **抽牌经济(关键): 起手8张, 之后每回合补2张。** 这是经模拟确认的节奏旋钮——补1会饿死重组玩法(57%回合卡手),补2才能让"桌面越大、链越长、倍率越高"的重组复利跑起来(均伤~41,后期还会爬升)。常量在 `arena_balance.json`。

## 1. 为什么会"一招制敌",以及为什么不上全局软上限

`base × mult` 是乘法叠乘,`base` 和 `mult` 同时拉满时是平方级增长,而血量是线性的——所以极限手能瞬间清场。

上一版提议过"全局单次伤害软上限(超过C后打折)"。**你否决了它,理由成立**:它对所有伤害隐性打折,玩家算的是 588 却只掉 290,像 bug,且削掉了打得好就更疼的正反馈。

**替代方案(本版采用)= 两个显式、可预期的系统机制:**

1. **阶段门**:Boss 分 N 段,**单次伤害不会击穿当前血量阈值溢出到下一段**。物理上保证"最少打 N 回合才能杀",且不打折任何在段内的伤害——透明、可预期。
2. **定额回复**:回合结束玩家**回固定值(默认+12),不再回满血**,让"存活"重新成为博弈主轴。

模拟证据(`outputs/arena_boss_sim.py` / `arena_boss_tune.py`):

| 验证项 | 结果 |
|---|---|
| 100血精英 · 无阶段门 | **13% 的回合直接秒杀**,最短1回合 |
| 100血精英 · 2阶段门 | **秒杀率 0%**,强制最少2回合 |
| 满血回复 | 胜率100%、**全程最低血就是满血**(存活毫无威胁,纯比DPS) |
| 定额回复+12 | 存活成为胜负主轴;敌伤越界即输 |

## 2. 系统级防秒三件套

### (1) 阶段门 — 防秒地板

- 每段 `hp_share` 决定段血量;`gating:true` 时单击有效伤害 = `min(伤害, 当前血 - 当前段血量下限)`,溢出蒸发(不结转、不反弹,除非该 Boss 另带 `overflow_reflect`)。
- 段切换时"强行结束本次伤害"= 你要求的设计:再大的爆发也只把这段打空,下一段从满状态重新算。
- 地板效应:**最少回合数 = 阶段数**。终 Boss 3 段 → 至少 3 回合,配合 `cap_hit_pct` 霸体进一步抬到 ~4。

### (2) 定额回复 — 存活压力

- `arena_balance.json → player.regen_per_turn`(默认12)、`regen_mode: fixed`。
- 平衡参考配置(扫描得到,贪心AI胜率约50%,真人会更高):**Boss 240血 / 2阶段 / 玩家80血定额回复12 / 敌方威胁 16(段1)→24(段2)**,7–9回合,0秒杀。
- 敌方威胁(threat)= 每段每回合对玩家的固定伤害,写在每段 `threat` 字段。**这是难度主旋钮**:威胁≤14基本稳赢,16→24 才是真打。

### (3) 存活遗物 / 技能 — 成长出口

回复≥16时几乎稳赢,所以**把"从50%推向稳赢"交给构筑**,这正是 roguelite 的 build 进度感:

- 存活遗物(`relics.json` category=survival):木盾(减伤)、石肤(-15%)、再起之风(+回复)、庇护符(每战免死1次)、阶段锚(段切换回血)、吸血牙。
- 技能(`skills.json`):壁垒(护盾)、重整旗鼓(救卡手)、破冰(解debuff)、拖延(延后蓄力斩)。

## 3. 机制目录(Boss 用来"干扰执行"的零件)

机制是数据对象,写在每段 `mechanics:[]` 里,可自由组合。完整字段与公式见《场面争夺_资产系统.md》。已用模拟标注了对玩家每手伤害的影响:

| 机制 type | 作用 | 对玩家均伤影响 |
|---|---|---|
| `resist` | 韧性:超过 soft 的部分打 k 折(局部、非全局) | 压大招,鼓励稳定输出 |
| `armor` | 固定减伤,惩罚碎手小链 | 小链净伤趋0,逼粗链 |
| `cap_hit_pct` | 霸体:单击 ≤ 最大血×pct(终Boss身份) | 保证最少回合 |
| `ban_pair` | 对子不计分 | **−13%**,卡手↑ |
| `nullify_shape` | 指定长度的链不计分 | **−10%** |
| `shape_mod` | 某类牌型分 halve/zero/−1(地狱犬三头) | 逼换牌型循环 |
| `dirty_color` | 弄脏一色,含该色的链 mult=0(预告+保底1) | mult 压 30–40% |
| `freeze_chain` | 冻结整条链:不可重组、不给新鲜倍率(⚠ 必须锁整条,不能只抽牌,否则会反向帮玩家) | 砍掉一条爆发线 |
| `charge` | 蓄力倒计时大招,达到 interrupt 伤害可打断 | 制造抢杀/转防 |
| `overflow_reflect` | 超量斩杀的溢出按 pct 反弹给玩家 | 罚"大锤敲钉子" |
| `pollute_deck` | 每回合往牌库塞污泥牌(面值0、不成链) | 有效手牌↓ |
| `single_target_no_splash` + `summon` | 群战,伤害只结算一个目标、溢出蒸发,定期补怪 | 罚浪费大招 |

> ⚠ **冻结的实现坑**:模拟里"抽走桌面N张"反而 +22% 伤害——因为抽牌改变了相邻链的签名、被误判成新鲜。真正的冻结必须**锁住整条链(标记 frozen,不参与重排、不计新鲜倍率),而不是把牌移出桌面**。

## 4. Boss 总表(9 个,数据见 `bosses.json`)

### 设计说明(逐个)

**cave_boar 洞穴野猪** — 最温柔的反爆发,用来教"韧性"这个词:大招会打折,所以两次中等 > 憋一次大的。复用现有美术。

**stone_pangolin 石壳穿山甲** — 护甲是"减法",专治每回合挤俩对子的碎手流;低 base 回合净伤趋近0。`pierce_at:100`:单击≥100 无视护甲,奖励粗链。和野猪正好相反(一个教别太大,一个教别太小)。

**cerberus 三头地狱犬** — 三段独立,每段封一类牌型,逼你每段换一套循环;储备牌要兼顾顺子与刻子两条路线。`shape_mod` 三种 op 各打击一种依赖。

**hyena_pack 拾荒鬣狗群** — `hp_segments` 多目标,伤害只落一个、溢出蒸发(`single_target_no_splash`),`summon` 每2回合补1只逼你别拖。把"大就是好"反过来:一个588在这只值40血。是"溢出蒸发"对照组(石像鬼是"溢出反弹")。

**swamp_hydra 沼泽九头蛇** — 第一个真正制造"抢杀 vs 转防"张力的 Boss。段2进倒计时,3回合斩140;那一回合打≥120可打断重置。读条把"龟缩养链"变成有代价的行为。

**rust_golem 锈蚀傀儡** — 最考验"重排"理解:预读下回合脏哪色,提前把强链建到安全色;刻子必含三色,弄脏任一色即废所有刻子倍率,把玩家从刻子流赶向纯色顺子流。`floor:1` 防归零死局。

**greedy_gargoyle 贪婪石像鬼** — 残血阶段要精算最后一击,习惯性憋大招收尾会被自己的溢出反杀。与鬣狗群构成"浪费 vs 受伤"对照。

**abyssal_polluter 深渊污染者** — 污染让时间站在 Boss 一边,反向逼加速;换牌排泥 vs 留着搏大链是核心资源决策。段2狂暴每回合2张泥。

**ancient_colossus 远古巨像** — 毕业考。常驻霸体(单击≤40%最大血≈136)做"不可被秒"保险;三段(各 hp_share 1/3)经阶段门保证**整场至少 3 回合**(实测最短 6),分别复用护甲/脏色/终局斩,逼你把三种应对串起来。段3带 2 回合蓄力斩(200),与"抢杀"形成最后对拼。实测胜率约 54%(贪心AI,真人更高)。

> 注:`stone_pangolin` 由单阶段改为 2 阶段,正是为了让阶段门生效——**单阶段敌人没有血量下限,会被一发大招秒掉**(实测单阶段时有 1.2% 秒杀率,改 2 阶段后归零)。这是给所有"精英/Boss"的通用规则:要么 ≥2 阶段,要么带 resist/cap 这类削峰机制。

## 5. 调参与接入

- **改平衡**:只动 `game/data/*.json`。常用旋钮:`arena_balance.json` 的 `regen_per_turn`(全局难度)、各 boss 的 `threat`(单怪难度)、`hp` / `hp_share`(节奏)、`mechanics` 参数(花样)。
- **加 Boss**:往 `bosses.json` 加一条;`ArenaDB` 自动校验(阶段 hp_share 合计=1、机制类型合法)。
- **接入代码**:`ArenaDB.new().load_all()` 读资产;`boss_phase_floors()` 给阶段门用;扣血处按"机制目录"逐条作用。详见《场面争夺_资产系统.md》的"接入 main.gd"。

## 6. 实现优先级

1. **阶段门 + 定额回复**(系统级,改"扣血前那一行"+回合结束回复)。做完"一招制敌"基本消失。
2. **三个低成本 Boss 验机制**:野猪(resist)、穿山甲(armor)、石像鬼(overflow_reflect)——都只改扣血前一行。
3. **中成本**:九头蛇(charge 倒计时)、锈蚀傀儡(dirty_color)——1个状态变量+预告文案+`_shape_score` 包一层。
4. **高成本**:地狱犬(三段+shape_mod)、鬣狗群(多血条)、污染者(污泥进 deck)、巨像(复合)。
5. **所有压制/打折务必 UI 明示**(倍率标签变灰、伤害飘字标"溢出蒸发/被吸收"),否则玩家当 bug。
